## üéì Core Concepts Recap

### The Three Categories

**üî¥ Category A: MUST ASK**
- Structural decisions affecting daily workflow
- Multiple valid approaches with tradeoffs
- Hard to change later
- Examples: Directory structure, workflow design

**üü° Category B: SHOULD SUGGEST**
- Preference decisions with working defaults
- Medium impact on user experience
- Can be adjusted over time
- Examples: Template formats, metadata schemas

**üü¢ Category C: AGENT DECIDES**
- Implementation details following established patterns
- Low/no user impact
- Technical choices
- Examples: Internal file structures, utility logic

---

### The Five Templates

1. **Multiple Choice** - 3-5 clear options with tradeoffs (Category A)
2. **Binary Decision** - Yes/No with recommendation (Category A/B)
3. **Preference Scale** - Spectrum from conservative to aggressive (Category A/B)
4. **Show & Tell** - Working example + alternatives (Category B)
5. **Open Exploration** - Understand user's mental model (Category A early)

---

### The Timing Strategy

**When to Ask**:
- ‚úÖ Session start (planning phase)
- ‚úÖ After analysis (context fresh)
- ‚úÖ After prototype (working example exists)
- ‚úÖ Before promotion (quality check)

**When NOT to Ask**:
- ‚ùå Mid-implementation (disrupts flow)
- ‚ùå During debugging (wrong context)
- ‚ùå After the fact (too late)
